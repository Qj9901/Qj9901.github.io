git使用技巧（自用）

字数总计:703|阅读时长: 2 分钟
| 阅读量:

近期在将源码保存到github、coding之类的git管理仓库，为之做个记录。

基本操作

  1. git init 仓库初始化

  2. git add <文件名> 添加文件到本地仓库暂存区

  3. git commit -m "<提交描述>" 提交到本地仓库

  4. git branch <分支名> 本地仓库创建分支

  5. git switch <分支名> 本地切换分支

    也可以直接用 git checkout -b <分支名> 表示创建并切换到分支

  6. git remote add <远程仓库名> <远程仓库地址> 关联/添加远程仓库

  7. git push <远程仓库名> <远程仓库分支> 提交到远程仓库

重命名git分支名称

  1. git branch -m <要改的本地分支名> <修改后的分支名> 修改本地分支

  2. git push origin :<远程修改前的分支名> 删除远程分支

  3. git push origin <修改后的分支名>:<修改后的分支名> push到远程分支

  4. git branch --set-upstream <修改后的分支名> <origin/修改后的分支名> 绑定远程分支

克隆指定分支/版本库

git clone -b <分支名> <远程仓库地址> 克隆制定分支

git clone -b <版本库> <远程仓库地址> 克隆指定版本库

git强制推送

git push <远程库名> <远程库分支> --force 强制推送
